### 模型架构: TCN-LSTM

```
Input (B, 168, 17)
  │
  ├── TCN Layer 1: CausalConv1d(17→64, k=3, d=1) + BN + ReLU + Residual
  ├── TCN Layer 2: CausalConv1d(64→64, k=3, d=2) + BN + ReLU + Residual
  ├── TCN Layer 3: CausalConv1d(64→64, k=3, d=4) + BN + ReLU + Residual
  │     感受野 = 1 + 2*(3-1)*(1+2+4) = 29 小时
  │
  ├── LSTM(input=64, hidden=128, layers=2, bidirectional=False)
  │     捕获长期时序依赖
  │
  ├── FC: Linear(128→64) + ReLU + Dropout(0.1)
  └── FC: Linear(64→24)
  │
Output (B, 24)   # 预测未来24小时充电负荷
```

**总参数量**: ~308K (轻量级, 适合联邦场景)

## 五、论文创新点提炼

### 核心创新 (3个卖点)

1. **VMD 信号分解预处理**  
   将充电负荷原始序列通过变分模态分解(VMD)拆分为高频+低频子序列, 降低预测难度。这在EV负荷预测领域尚未被广泛应用。

2. **TCN-LSTM 混合架构**  
   TCN 通过空洞因果卷积提取多尺度局部模式, LSTM 捕获长期依赖, 结合两者优势。相比单纯LSTM, 训练更稳定、梯度不易消失。

3. **谱聚类联邦学习 (Clustered FL)**  
   提出基于模型参数相似度的谱聚类方法, 将负荷模式相似的站点分组, 簇内独立聚合。完美解决Non-IID问题, 是当前FL领域的研究热点。

### 论文对比实验设计 (5组)

| 编号 | 方法 | 说明 |
|------|------|------|
| M1 | Local-only | 每站仅用自身数据训练, 孤岛模式 baseline |
| M2 | Centralized | 所有数据集中训练, 理论上限 (不保护隐私) |
| M3 | FedAvg + LSTM | 基础联邦学习 baseline |
| M4 | FedProx + TCN-LSTM | 本文方法 v1: 加近端正则化 |
| M5 | Clustered FL + VMD + TCN-LSTM | **本文完整方法** |

### 评估指标
- **RMSE** (均方根误差): 对突发峰值敏感
- **MAE** (平均绝对误差): 反映整体偏差
- **MAPE** (平均绝对百分比误差): 消除站点规模差异

## 九、特征选择分析结论

### 分析方法
1. **Spearman 秩相关系数** — 捕捉非线性单调关系 (优于 Pearson)
2. **互信息 (MI)** — 捕捉任意类型的统计依赖 (含离散变量)

### 筛选标准
保留条件: Spearman |ρ| > 0.05 **或** MI > 中位数 (0.21)

### 最终选择的 16 个特征 (全部保留)

| 特征 | 类型 | Spearman ρ | MI | 保留原因 |
|------|------|-----------|-----|---------|
| hour_sin | 时间循环编码 | +0.529 | 0.600 | 强相关: 充电高峰有明显日内周期 |
| hour_cos | 时间循环编码 | +0.385 | 0.491 | 强相关: 与 hour_sin 正交配对 |
| solarradiation | 气象 | -0.458 | 0.429 | 强相关: 白天太阳辐射强时充电少 |

| e_price | 价格 | +0.105 | 0.851 | Spearman 弱但 MI 最高: 电价与负荷有复杂非线性关系 |

### 调参优先级和策略

#### 第1优先级: 学习率 lr (影响最大)

| 现象 | 诊断 | 调整方向 |
|------|------|---------|
| Loss 不下降, 震荡剧烈 | 学习率太大 | lr 减半: 1e-3 → 5e-4 |
| Loss 下降极慢, 50轮还没收敛 | 学习率太小 | lr 加倍: 1e-3 → 2e-3 |
| Loss 快速下降后不再变化 | 已收敛, 可能欠拟合 | 尝试增大模型容量 |
| Loss 下降后又上升 | 过拟合或lr太大 | lr 减小 + 加 weight_decay |

**推荐搜索范围**: `[5e-4, 1e-3, 2e-3]`

#### 第2优先级: 本地训练轮次 local_epochs

| 现象 | 诊断 | 调整方向 |
|------|------|---------|
| 各站点 RMSE 差异极大 | 客户端漂移(drift) | 减少: 5 → 3 |
| 全局模型不如本地模型 | 本地训练不足 | 增加: 5 → 8 |
| 模型对某些站点特别差 | Non-IID 问题严重 | 改用聚类联邦 |

**核心原理**: local_epochs 越大, 本地模型越适应自己数据, 但越偏离全局模型。  
**推荐**: 先用 3-5, 配合 FedProx 的 mu 参数一起调。

#### 第3优先级: FedProx 近端系数 mu

| 设置 | 效果 |
|------|------|
| mu = 0 | 退化为 FedAvg, 无约束 |
| mu = 0.001 | 弱约束, 允许本地较大自由度 |
| mu = 0.01 | 适中约束 (推荐起始值) |
| mu = 0.1 | 强约束, 本地模型被拉回全局 |
| mu = 1.0 | 过强, 本地几乎不学习 |

**调参逻辑**: 如果客户端漂移严重 (各站 RMSE 差异大), 增大 mu; 如果全局模型效果太差, 减小 mu 给本地更多空间。

#### 第4优先级: 输入窗口 seq_len

| 设置 | 含义 | 适用场景 |
|------|------|---------|
| 48 | 2天历史 | 数据量少时, 或负荷模式简单 |
| 72 | 3天历史 | 平衡选择 |
| 168 | 7天历史 (推荐) | 能捕捉完整周模式 |
| 336 | 14天历史 | 数据充足时, 捕捉更长期模式, 但训练慢 |

**注意**: seq_len 越长, 训练越慢, 显存占用越大。RTX 4060 (8GB) 建议不超过 336。

#### 第5优先级: 聚类数 n_clusters

| 设置 | 适用情况 |
|------|---------|
| 2 | 站点数少 (<50), 或只有两种明显模式 (商业/住宅) |
| 3 | 推荐起始值, 适合多数情况 |
| 5 | 站点类型多样, 如同时有商业/住宅/交通枢纽/工业/旅游区 |
| 8+ | 不推荐, 簇太多接近本地训练, 失去联邦优势 |

**判断方法**: 看 `station_comparison.png` 中各站 RMSE 的分布, 如果有明显的 2-3 组, 就用对应的 n_clusters。

#### 第6优先级: 模型结构参数

| 参数 | 调小 | 调大 |
|------|------|------|
| lstm_hidden (默认128) | 减少过拟合, 加速训练 | 增加模型容量, 可能提升精度 |
| tcn_channels (默认[64,64,64]) | 简化模型 | 增加层数或通道: [64,128,128] |
| tcn_dropout (默认0.2) | 模型欠拟合时减小 | 过拟合时增大至 0.3 |

**注意**: 模型参数修改需要直接编辑 `config.py` 文件, 不支持命令行传参。
